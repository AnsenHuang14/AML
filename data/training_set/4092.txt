Notepad++ 再戳中國玻璃心，但仍沒有引起 GitHub 官方出手
不少人寫程式或是日常文書處理會用強化版本的純文字編輯器，而在 Windows 平台聞名的編輯器 Notepad++，開發者再次戳中國人的敏感神經，7.8.1 版本命名為解放維吾爾人 (Free Uyghur)，issue 頁面滿布用中文謾罵開發者的言論，網軍攻擊官網導致無法連線情形。Notepad++ 開發者 Don Ho 常在開發專案之外，透過專案發布新版本機會表達政治立場。這次 7.8.1 版本則是聲援新疆維吾爾人，命名為解放維吾爾人，要大家正視大批維吾爾人被迫關進再教育營。如預期專案 GitHub 頁面湧入不滿的聲音，issue 頁面滿滿用中文謾罵開發者的話語，Notepad++ 網站也面對 DDoS 攻擊，被迫需要求助 Cloudflare 來清理流量。從程式碼代管平台 SourgeForge 到現今為微軟一員的 GitHub 上面，Notepad++ 開發者始終不願意讓中國人心情舒暢，常常在多次喚醒大家政治意識時，採到中國敏感神經。先前 2008 年北京奧運前夕，Notepad++ 在 SourgeForge 專案首頁懸掛杯葛北京奧運的橫幅廣告，導致好幾個天時間中國用戶無法連到 SourgeForge。2014 年 6 月 4 日發布 Notepad++ 6.6.4 刻意玩弄版本號碼的巧合，命名為 64 天安門版本。另外 Notepad++ 繁中語言版本用中華民國國旗代表引起不少中國人不悅。Notepad++ 不只招惹中國人，2015 年 1 月在我是查理 (Je suis Charlie) 爭取言論自由的行動當中，Notepad++ 藏了聲援的彩蛋。而有駭客團體 Fallaga Team 不滿該彩蛋，駭進 Notepad++ 網站。不過相比 GitHub 接受西班牙憲兵要求，將加泰隆尼亞獨立運動團體傳遞抗議活動的 App APK 檔移除，惹來牽制言論的爭議。Notepad++ 的紛爭相比之下，大概是在中國人還有開發者之間小打小鬧的情形，GitHub 大概不會介入。不過以 GitHub 在開源開發界的地位，只會有越來越多令微軟立場尷尬的時機，動手封鎖特定專案，恐怕會像這次應西班牙政府要求而被人非議的狀況。中國這邊因為不少開發者仰賴 GitHub 的開源資源，儘管有像是趙家人名冊令中國官方不悅的資料，動手封鎖 GitHub 將導致程式開發圈停擺，不大可能全站被中國防火長城封鎖。
